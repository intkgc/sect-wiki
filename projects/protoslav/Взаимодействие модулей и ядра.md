Все модули запускаются в виде сторонних процессов. Все процессы общаются между собой через пересылку сообщений. Для сообщений используется gRPC, Protobuffers, а так же 

Для обращений к ядру используются сообщения на основе Protobuffer, в то время как модули могут регистрировать сообщения только на основе json схемы. 

## Сообщения ядра

Сообщения ядра можно разделить на три функциональные группы:
- инфраструктурные сообщения
- сообщения для работы с нейросетью
- сообщения для работы с памятью

### Инфраструктурные сообщения

Инфраструктурные сообщения отвечают за динамическую регистрацию модулей, за отправку о статусе работы модуля, за динамическую регистрацию сообщений модулей, а так же за отправку динамического сообщения модулем.

### Сообщения для работы с нейросетью

Эти сообщения позволяют модулям отправлять запросы LLM 

### Сообщения для работы с памятью

Эти сообщения предназначены для работы с файловой системой, долговременной и кратковременной памятью ИИ агента


## Сообщения модулей

Модули могут регистрировать свои собственные сообщения через json схему, которая затем отправляется в ядро через инфраструктурное сообщение о регистрации схемы


Схема содержит в себе имена полей, а так же их тип и позицию в массиве байт.

Вот пример схемы

```json
{
	"return" : {
		"type" : "array",
		"data": "string"
	},
	
	"from" : {
		"type": "string",
		"layout": 0
	},
	
	"to" : {
		"type": "string",
		"layout": 1
	}
}
```
